# Dicionário de Linguagem Ubíqua

| Termo                  | Significado                                              | Exemplo                                     |
|------------------------|----------------------------------------------------------|---------------------------------------------|
| **Usuário**            | Pessoa que utiliza o sistema ou aplicação.               | O usuário fez login no sistema.              |
| **Autenticação**       | Processo de verificação da identidade do usuário.        | A autenticação é necessária para acessar.   |
| **API**                | Interface de Programação de Aplicações.                  | A aplicação se comunica com a API externa.  |
| **Backend**            | Parte de um sistema que lida com a lógica e o processamento de dados. | O backend processa as solicitações do frontend. |
| **Frontend**           | Parte de um sistema visível para o usuário, com a qual ele interage. | O frontend exibe a interface de usuário.   |
| **Framework**          | Conjunto de ferramentas e convenções para facilitar o desenvolvimento. | O desenvolvedor utiliza um framework para agilizar o trabalho. |
| **Banco de Dados**     | Sistema organizado para armazenar, recuperar e gerenciar dados. | Os dados do aplicativo são armazenados no banco de dados. |
| **Deploy**             | Processo de lançamento de uma aplicação para um ambiente de produção. | Após a conclusão do desenvolvimento, realizamos o deploy da aplicação. |
| **Feedback**           | Resposta ou retorno fornecido ao usuário ou desenvolvedor. | O usuário forneceu feedback sobre a experiência do aplicativo. |
| **UI/UX**              | Design de Interface do Usuário/Experiência do Usuário.   | O designer melhorou a UI/UX da aplicação.  |
| **Sprint**             | Período de tempo fixo para o desenvolvimento de um conjunto de funcionalidades. | Estamos na última sprint do ciclo de desenvolvimento. |
| **Scrum**              | Metodologia ágil para gerenciamento de projetos.          | Adotamos o Scrum para organizar as tarefas. |
| **Git**                | Sistema de controle de versão distribuído.               | O código-fonte é gerenciado usando o Git.   |
| **Merge**              | Combinação de alterações de uma branch com outra.        | Após revisão, realizamos o merge das alterações na branch principal. |

Este é apenas um exemplo. Personalize-o de acordo com os termos específicos e exemplos relevantes para o contexto do seu projeto ou equipe.
